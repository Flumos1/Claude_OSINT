# Claude OSINT Workspace

Рабочее пространство для OSINT-задач: корпоративный due diligence, threat intelligence,
расследования/фактчекинг и обучение.

**Географический приоритет: 🇺🇦 Украина.** Россия поддерживается; другие страны
(🇨🇿 Чехия, 🇵🇱 Польша, 🇩🇪 Германия, 🇺🇸 США…) добавляются по шаблону. Источники
организованы страново в [knowledge/sources/](knowledge/sources/) — по умолчанию начинай с
[ua.md](knowledge/sources/ua.md). Как добавить страну: [knowledge/sources/README.md](knowledge/sources/README.md).

## Назначение и рамки (читать первым)

Этот workspace — для **легального и этичного** OSINT по открытым источникам:
- проверка компаний, контрагентов, бенефициаров, аффилированности;
- threat intelligence: мониторинг утечек, фишинга, инфраструктуры угроз, защита своего бренда;
- расследовательская журналистика и верификация фактов;
- обучение методологии и работе с инструментами; CTF.

**Не делаем:** слежку и преследование частных лиц, деанонимизацию/доксинг ради вреда,
обход авторизации и взлом, покупку краденых данных, нарушение ToS/закона ради цели.
Если задача дрейфует в эту сторону — остановись и проговори это с пользователем.
Этика и право: [knowledge/ethics-legal.md](knowledge/ethics-legal.md).

## Как Claude работает в этом проекте

1. **Любая задача начинается с цели и авторизации.** Что за объект, зачем проверяем,
   есть ли правовое основание. Если объект — частное лицо, проверь рамки выше.
2. **Используй скилы** из `.claude/skills/` под конкретный тип объекта (компания, домен,
   персона, угроза). Скил = пошаговый плейбук + чеклист источников.
3. **Веди кейс** в `cases/<slug>/` по структуре из `cases/_TEMPLATE/`. Всё собранное —
   с источником, датой, ссылкой (provenance). Без источника факта не существует.
4. **Разделяй факт / вывод / гипотезу.** Маркируй уверенность. Не выдавай корреляцию
   за подтверждение. Совпадение имени ≠ тот же человек.
5. **Отчёт** генерируй по шаблону из `templates/` через скил `osint-report`.
6. **OPSEC:** помни про цифровой след самого исследователя — [knowledge/opsec.md](knowledge/opsec.md).

## Структура

```
.claude/skills/   — Claude-скилы (плейбуки): company-dd, person-osint,
                    domain-infra, threat-intel, osint-report, osint-orchestrator
knowledge/        — методология, sources/ (источники по странам: ua/ru/intl), инструменты, этика, OPSEC
templates/        — шаблоны отчётов
scripts/          — утилиты автоматизации (Python/PowerShell)
cases/            — расследования; _TEMPLATE — структура нового кейса
data/             — сырые выгрузки (gitignore-кандидат, может содержать чувствительное)
```

## Принципы качества (золотые правила OSINT)

- **Provenance** — каждый факт со ссылкой, скриншотом/архивом и датой сбора.
- **Минимизация** — собираем только то, что нужно для цели; не «всё подряд».
- **Верификация** — минимум два независимых источника на ключевой факт.
- **Архивируй** — открытые данные исчезают: сохраняй в Wayback/локально на момент сбора.
- **Точность дат** — фиксируй дату сбора; данные реестров устаревают.
- **Безоценочность** — отделяй наблюдаемые факты от интерпретаций.
