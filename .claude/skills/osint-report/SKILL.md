# OSINT Report

Превращает собранный кейс в структурированный отчёт. Шаблоны — в [templates/](../../../templates/).

## 0. Выбор шаблона

| Тип | Шаблон |
|-----|--------|
| Компания/контрагент | templates/company-report.md |
| Персона | templates/person-report.md |
| Домен/инфраструктура | templates/domain-report.md |
| Threat / инцидент | templates/threat-brief.md |

## Принципы оформления

1. **Provenance в каждом факте.** Источник + ссылка + дата сбора. Утверждение без
   источника не включаем (или явно помечаем как гипотезу).
2. **Маркируй достоверность** ключевых фактов: [A1]..[F6] (см. methodology).
3. **Разделяй** ФАКТ / ВЫВОД / ГИПОТЕЗА. Не выдавай корреляцию за подтверждение.
4. **Executive summary первым** — главные выводы и риски на 5–10 строк для занятого читателя.
5. **Оценка рисков** — где уместно, таблица: риск / уровень / обоснование / источник.
6. **Безоценочность** в фактах, аккуратность в выводах; уважай презумпцию.
7. **Воспроизводимость** — приложи методологию/журнал, чтобы путь можно было повторить.
8. **Минимизация** — в отчёт только релевантное цели; лишние частные данные исключаем.

## Структура (общая)

1. Шапка: объект, заказчик/цель, период, исполнитель, дата, дисклеймер.
2. Executive summary + сводка рисков.
3. Методология и источники (кратко).
4. Фактура по разделам (со ссылками и [оценкой]).
5. Анализ связей (граф/таблица сущностей).
6. Выводы и рекомендации.
7. Приложения: скриншоты/архивы, журнал сбора, список источников.

## Выгрузка

- Markdown — по умолчанию в `cases/<slug>/report.md`.
- DOCX — через скил `anthropic-skills:docx` (для передачи заказчику).
- PDF — через скил `anthropic-skills:pdf`.
- Таблица сущностей/IOC/рисков — через `anthropic-skills:xlsx`.

## Дисклеймер (включать в отчёт)

> Отчёт основан на открытых источниках на дату сбора; данные могли измениться.
> Сведения не являются юридическим заключением. Выводы носят вероятностный характер
> и требуют подтверждения для юридически значимых решений.
