# Person OSINT

> **Стоп-контроль.** Перед стартом проверь основание по [knowledge/ethics-legal.md](../../../knowledge/ethics-legal.md).
> Допустимо: KYC/комплаенс, бенефициар компании, публичная фигура в расследовании в
> общественных интересах, согласие субъекта, проверка деловой репутации при найме (с основанием).
> Недопустимо: слежка за частным лицом без основания, доксинг, преследование, несовершеннолетние.
> Если основания нет — откажись и объясни.

## 0. Постановка

Зафиксируй: кого проверяем, **зачем**, какое основание, какие IR. Минимизация:
собирай только релевантное цели, не «всё о человеке».

## ⚡ Инструмент: интеллектуальный поиск ФЛ

Оркестратор `scripts/person_search.py` (и раздел «Поиск ФЛ» в веб-платформе) делает фан-аут
по открытым источникам сразу:
```
python scripts/person_search.py "Прізвище Ім'я По-батькові" --countries ua,ru,intl \
    --dob 1980-01-01 --rnokpp <код> --email <e> --username <u>
```
Что делает: генерирует **варианты написания** (укр/рус/латиница + перестановки) для полноты;
валидирует РНОКПП/ИНН; **живой** запрос в НАЗК (декларації); пивотит переданные email/username;
даёт точные deep-ссылки на реестры UA/RU/межд. (суди, борги, санкції, PEP, ФОП, OpenSanctions,
ICIJ, Aleph). **Только легально:** без «пробива»/слитых баз; паспорт и недвижимость-по-ФИО закрыты
(см. правовые оговорки в выводе). Дальше — ручная проверка по ссылкам и верификация атрибуции.

## 1. Селекторы

Стартовые селекторы и как разворачивать (см. [knowledge/methodology.md](../../../knowledge/methodology.md)):
**ФИО, email, username, телефон, фото.** Держи варианты транслитерации (Ivanov/Ivanoff)
и неполные/девичьи фамилии.

## 2. Имя → присутствие

- WebSearch: `"Имя Фамилия"` + контекст (город/компания/профессия), в кавычках, на RU и EN.
- Соцсети: VK (vk.com поиск people), OK, Telegram (tgstat), LinkedIn, X, Facebook, Instagram.
- Профессиональный след: публикации, выступления, патенты (Роспатент/Google Scholar), GitHub.
- Реестры (если применимо к цели): ФССП (долги), ЕФРСБ (банкротство физлица), суды (sudrf),
  ЦИК (если публичная фигура/кандидат), дисквалификация.

## 3. Email → аккаунты

- HIBP — в каких утечках засветился (передай в **threat-intel** для деталей).
- Holehe/EmailRep — где зарегистрирован (по логике «забыли пароль»).
- Gravatar по хешу email; домен email (личный/корпоративный) → whois.
- Email из git-коммитов (GitHub) часто связывает ник с реальным именем.

## 4. Username → платформы

- WhatsMyName / Sherlock / Maigret — один ник на множестве сайтов.
- Переиспользование ника — мост между платформами, но осторожно: ники бывают чужими.
  Верифицируй принадлежность (аватар, стиль, перекрёстные ссылки).

## 5. Телефон

- Формат/оператор/регион (numverify, PhoneInfoga).
- Привязка к мессенджерам (наличие в Telegram/WhatsApp — только публичное отображение).
- Объявления (Avito/Юла) — если телефон публиковался.

## 6. Фото / изображения

- Reverse image: Yandex (силён на лицах, особенно RU), Google Lens, TinEye, Bing.
- EXIF (ExifTool) если не вычищен; фон → гео/хронолокация (см. methodology).

## 7. Связи и аффилированность

- Если человек — бенефициар/директор → его компании через **company-dd** (Checko/Rusprofile).
- Семейные/деловые связи только из открытых публичных источников и под цель.
- Граф связей в `entities.md`.

## 8. Санкции / PEP

OpenSanctions, OFAC, нац. списки — особенно для KYC/комплаенс.

## Дисциплина атрибуции

- Совпадение имени/ника/фото ≠ тот же человек. Требуй ≥2 независимых подтверждения связи.
- Маркируй достоверность ([A1]..[F6]) и уровень (ФАКТ/ВЫВОД/ГИПОТЕЗА).
- Не публикуй частные данные; в отчёте — только релевантное цели.

## Выход

Кейс + отчёт через **osint-report** ([templates/person-report.md](../../../templates/person-report.md)).
