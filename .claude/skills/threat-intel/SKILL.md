# Threat Intelligence

Защитный OSINT: утечки, фишинг, IOC, мониторинг бренда. Источники:
[knowledge/sources/intl.md](../../../knowledge/sources/intl.md). Подозрительное — в изоляции
([knowledge/opsec.md](../../../knowledge/opsec.md)).

## 0. Контекст

Что защищаем/расследуем: свой домен/бренд? инцидент? проверка индикатора?
Для утечек — работаем только со **своими** или авторизованными данными (см. этику).

## 1. Утечки учётных данных

- **HIBP** (haveibeenpwned.com) — email/домен в известных breach'ах: какие утечки, что слито.
  Domain search показывает все засвеченные адреса своего домена.
- IntelX — упоминания в leaks/пастах/даркнете (free/paid).
- **Не покупаем** слитые пароли и не используем их для доступа. Цель — узнать факт
  компрометации и инициировать смену паролей/мониторинг, а не эксплуатацию.

## 2. Мониторинг бренда / фишинг / тайпсквоттинг

- Сгенерируй типосквот-варианты домена (dnstwist-логика: замены/перестановки/омоглифы,
  другие TLD) и проверь регистрации (whois) и активность (urlscan, VirusTotal).
- crt.sh по бренду — сертификаты на домены, мимикрирующие под бренд.
- WebSearch + соцсети: фейковые аккаунты/страницы, использующие бренд/логотип.
- Свежезарегистрированный похожий домен + копия дизайна = активный фишинг → в **domain-infra**
  для атрибуции инфраструктуры.

## 3. Анализ IOC

| IOC | Где проверять |
|-----|---------------|
| IP | VirusTotal, AbuseIPDB, GreyNoise, Shodan, OTX |
| Домен/URL | VirusTotal, urlscan, URLhaus, ThreatFox |
| Файл/хеш | VirusTotal, MalwareBazaar |
| Email | HIBP, EmailRep |

Собирай контекст: связанные IOC (пассивный DNS, общий сертификат/инфраструктура),
кампания/семейство (OTX-пульсы), TTP. Привязывай к ATT&CK при наличии.

## 4. Поверхность атаки своей организации

- Поддомены/забытые сервисы → **domain-infra** (crt.sh, subfinder).
- Открытые сервисы → Shodan/Censys по своим IP/ASN.
- Утёкшие секреты в публичном коде: GitHub dorks, gitleaks/trufflehog по своим репозиториям/упоминаниям.
- Публичные документы с метаданными (внутр. имена, ПО) — Google `filetype:` по своему домену.

## 5. Даркнет / пасты / форумы

- IntelX, публичные пасты (psbdmp) — упоминания домена/бренда/email.
- Только наблюдение открытых источников; не вступаем в нелегальные транзакции.

## 6. Реагирование (output)

- Список индикаторов + достоверность + рекомендация (блокировка, тейкдаун, смена паролей,
  мониторинг, уведомление).
- Для тейкдауна фишинга — данные регистратора/хостера (abuse-контакт из whois).

## Выход

Кейс + бриф через **osint-report** ([templates/threat-brief.md](../../../templates/threat-brief.md)).
Формат брифа: TL;DR → индикаторы → анализ → рекомендации → IOC-таблица.
