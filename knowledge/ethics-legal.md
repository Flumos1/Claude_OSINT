# Этика и право OSINT

> Это рабочая памятка, а не юридическая консультация. При реальном риске — консультируйся с юристом.

## Базовый принцип

OSINT работает только с **открытыми, законно доступными** данными и **законными** методами.
«Доступно технически» ≠ «законно собирать и использовать». Цель не оправдывает метод.

## Красные линии (не делаем)

- **Несанкционированный доступ:** подбор/использование чужих паролей, обход авторизации,
  эксплуатация уязвимостей, доступ к непубличным системам. Это уже не OSINT, а взлом.
- **Покупка/использование краденых данных:** «пробивы», слитые базы для деанона частных лиц.
- **Социальная инженерия с обманом** ради получения данных (претекстинг под чужое имя/должность).
- **Доксинг и преследование:** публикация частных данных для травли, запугивания, мести.
- **Слежка за частными лицами** без правового основания (партнёр, бывший, сосед, активист).
- **Нарушение закона о перс. данных** при сборе/хранении/передаче (152-ФЗ в РФ, GDPR в ЕС).
- **Обход ToS** там, где это создаёт юридический/договорный риск (массовый скрейпинг под фейками).

## Когда работа допустима

- Есть **законная цель**: комплаенс/KYC, проверка контрагента, защита своих активов и бренда,
  расследование в общественных интересах, согласие субъекта, исполнение договора/закона.
- **Минимизация:** собираешь только релевантное цели, не «досье на всякий случай».
- **Пропорциональность:** глубина вторжения в приватность соразмерна важности цели.
- **Хранение:** чувствительные данные защищены, срок хранения ограничен, доступ — по нужде.

## Правовые рамки (ориентиры)

- **РФ:** 152-ФЗ «О персональных данных»; ст. 137 УК (неприкосновенность частной жизни);
  ст. 272 УК (неправомерный доступ к компьютерной информации); 149-ФЗ (информация).
- **ЕС:** GDPR — законное основание обработки, права субъекта.
- **США:** CFAA (доступ к компьютерам), FCRA (если для скрининга/найма).
- **Везде:** клевета/диффамация — выводы должны опираться на факты с источниками.

## Особые категории (повышенная осторожность)

- Здоровье, религия, политвзгляды, сексуальная ориентация, биометрия — спец. категории
  перс. данных, повышенный риск. Без явной необходимости и основания не собираем.
- **Несовершеннолетние** — отдельная защита, по умолчанию не объект сбора.

## Чеклист перед стартом кейса

- [ ] Сформулирована законная цель и основание.
- [ ] Объект — компания/публичная фигура/инфраструктура, а не частное лицо без оснований?
- [ ] Методы — только пассивные/открытые, без обхода авторизации?
- [ ] Минимизация: собираю только нужное под IR?
- [ ] План хранения и удаления чувствительных данных?
- [ ] Выводы будут опираться на источники, без домыслов?

Если хоть один пункт «нет» — остановись и пересмотри.

## Профессиональные кодексы (полезно знать)

- Принципы Bellingcat / GIJN для расследовательской журналистики: прозрачность метода,
  верификация, минимизация вреда, право на ответ.
- SANS/intelligence community: разделение факта и оценки, цепочка provenance.
