# OPSEC исследователя

> Цель — не оставить след у объекта и не скомпрометировать расследование/себя.
> Уровень мер соразмерен чувствительности кейса. Для рутинного DD по компании
> достаточно базовой гигиены; для чувствительных расследований — полный набор.

## Зачем

- Объект не должен узнать, что им интересуются (это меняет его поведение, уничтожает следы,
  иногда опасно для исследователя).
- Не «засветить» свою личность, организацию, локацию через цифровой след запросов.
- Не заразиться/не скомпрометировать систему при работе с подозрительной инфраструктурой.

## Сетевая гигиена

- **Отдельный браузер/профиль** для расследований; чистка cookies, без логина в личные аккаунты.
- **VPN / выделенный IP** — не вести разведку с домашнего/рабочего IP, особенно по
  инфраструктуре, которая логирует визиты. Учитывай юрисдикцию exit-ноды.
- **Не кликай напрямую** по ссылкам объекта без изоляции: используй urlscan.io, кэш,
  Wayback, песочницу. Прямой заход = запись в их логах + риск деанона/малвари.
- **Sock puppet** (исследовательский псевдо-аккаунт) для соцсетей — отдельная личность,
  своя почта/телефон, прогретая история. Никогда не смешивать с личным аккаунтом.
  Создание — в рамках ToS платформы и закона; не для обмана конкретных жертв.

## Изоляция и безопасность системы

- Работа с подозрительными файлами/сайтами — в **ВМ/песочнице** (отдельная ОС, snapshot).
- Не открывай документы объекта на рабочей машине; анализируй метаданные в изолированной среде.
- Антивирус/обновления; не запускай неизвестные бинарники.

## Активный vs пассивный сбор

- **Пассивный** (предпочтительно): архивы, кэши, пассивный DNS, сторонние агрегаторы —
  объект не видит запрос.
- **Активный** (whois-запрос, прямой заход, port-scan, отправка сообщения) — оставляет след
  и может быть незаконным (сканирование). Делай осознанно и только при основании.

## Атрибуция запросов

- Поисковики и сервисы логируют запросы; некоторые реестры показывают, кто смотрел.
- LinkedIn и подобные уведомляют о просмотрах профиля — отключай публичность/режим инкогнито.
- Reverse-image и upload-сервисы могут хранить загруженное — не грузи чувствительное.

## Гигиена данных кейса

- Чувствительные выгрузки — в `data/`, не в общий доступ; рассмотри шифрование контейнера.
- Хеши файлов-доказательств для целостности; журнал происхождения.
- По завершении — политика удаления по `knowledge/ethics-legal.md`.

## Чеклист OPSEC (для чувствительного кейса)

- [ ] Отдельный профиль/браузер, без личных логинов.
- [ ] VPN/выделенный канал, проверен на утечки DNS/WebRTC.
- [ ] Подозрительные ссылки — только через изоляцию (urlscan/Wayback/песочница).
- [ ] Sock puppet (если нужен) отделён от личности.
- [ ] Файлы объекта — в ВМ.
- [ ] Журнал действий ведётся; данные защищены.
