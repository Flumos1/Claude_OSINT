# Платформа Claude OSINT — архитектура v2 (план)

Эволюция текущей FastAPI+SPA веб-оболочки во всеобъемлющую search-first платформу
с подсказками («что дальше»), оценкой уверенности и асинхронными скан-джобами.

> Статус: план. Решения приняты 2026-06-29. Реализация поэтапная (см. Дорожную карту).

## Принятые решения

| Вопрос | Решение |
|--------|---------|
| Фронтенд | **React + Vite + TypeScript + Tailwind + shadcn/ui** (вместо vanilla SPA) |
| Бэкенд | FastAPI (как сейчас) поверх движка энричеров `scripts/` |
| Развёртывание | Локаль (127.0.0.1) сейчас + **готовность к деплою**: auth, мульти-юзер, Docker |
| Первая итерация | **Умный поиск + подсказки** |

## Информационная архитектура

Платформа становится **search-first**: единое поле с авто-детектом типа сущности.

| Раздел | Назначение |
|--------|-----------|
| Поиск | Единый ввод → авто-детект (email/домен/IP/телефон/ник/ЄДРПОУ/ИНН/ФИО) → энричеры. ⌘K-палитра, история |
| Результат-канвас | Вкладки: Находки (с %-уверенностью), Граф, Сущности, Таймлайн, JSON, Экспорт + панель подсказок |
| Граф | Сквозной граф связей по кейсам (vis-network / sigma.js) |
| Поиск ФЛ | Гайдед-форма с обязательным правовым основанием |
| Инструменты | Индекс 1400+ (`tools-index.json`) + curated, фильтры, этик-флаги |
| Источники / Скилы / Кейсы | По странам; плейбуки; менеджмент расследований |
| Настройки | API-ключи (UI вместо .env), тумблеры deep-режима, тема |

## Три опорные идеи

1. **Подсказки (suggestion engine).** После результата — контекстные карточки следующего
   шага по правилам: `email найден → проверить утечки`, `GitHub → собрать репо/секреты`,
   `username низкой уверенности → запустить deep`, `готово → сохранить в кейс`. Правила
   декларативны (тип входа + находки → предложения), расширяются вместе с энричерами.
2. **Уверенность везде.** Скоринг 0–100% + Admiralty (D3/D4/D5) бейджами на каждой находке
   (см. `scripts/enrichers/username_enr.py` как референс). Отделяет факт от гипотезы.
3. **Асинхронные джобы.** Долгие сканы (deep username 740 сайтов ~60 c) — фоновый воркер +
   прогресс через SSE/WebSocket: «проверено 340/740…». Основа масштабируемости.

## Технические заметки

- **Бэкенд-джобы:** очередь задач (стартово — `asyncio` + in-memory store; при деплое →
  Redis/RQ или Celery). Эндпоинты: `POST /api/jobs` (старт), `GET /api/jobs/{id}` (статус),
  `GET /api/jobs/{id}/stream` (SSE прогресс).
- **Auth (deploy-ready):** заложить middleware и модель пользователя сразу, по умолчанию
  выключено для локали. Данные расследований чувствительны — наружу только за авторизацией.
- **Структура фронта:** `web/ui/` (Vite-проект), сборка в `web/static/dist/`, FastAPI раздаёт.
  Dev: Vite-proxy на FastAPI. Prod: один контейнер.
- **Дизайн-язык:** тёмная-first «intelligence console», Inter + JetBrains Mono для данных,
  граф как холст, цвет = семантика (уверенность/риск), без градиентов, клавиатура-first, адаптив.

## Дорожная карта

1. ✅ **Умный поиск + подсказки**: авто-детект типа, результат-канвас с бейджами
   уверенности, декларативный движок подсказок. На React+Vite, поверх `/api/enrich`.
2. ✅ **Асинхронные джобы + прогресс:** deep-скан как джоба (`web/jobs.py`) с SSE
   (`/api/jobs`, `/api/jobs/{id}/stream`); прогресс-бар в UI; `stream_username` в энричере.
3. **Каталог инструментов 1400+:** поиск/фильтр/этик-флаги по `tools-index.json`.
4. **Менеджмент кейсов + сквозной граф + экспорт** (MD/DOCX/PDF через скилы).
5. **Deploy-пакет:** auth, мульти-юзер, Docker, reverse-proxy.
